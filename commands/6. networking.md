### **Linux Networking Commands**  
Linux provides powerful networking tools to configure, troubleshoot, and monitor network connections. Here are the most essential networking commands for DevOps and system administrators:  

---

## **1. `ifconfig` – Interface Configuration**  
- **View Network Interfaces and IP Addresses:**  
  ```bash
  ifconfig
  ```
- **Bring an Interface Up or Down:**  
  ```bash
  ifconfig eth0 up
  ifconfig eth0 down
  ```
- **Assign IP Address to an Interface:**  
  ```bash
  ifconfig eth0 192.168.1.10 netmask 255.255.255.0
  ```
- **Example:**  
  ```bash
  ifconfig eth0 192.168.1.100 netmask 255.255.255.0 up
  ```

> **Note:** `ifconfig` is deprecated on newer Linux distributions. Use `ip` instead.

---

## **2. `ip` – Modern Replacement for `ifconfig`**  
- **Show All Network Interfaces and IP Addresses:**  
  ```bash
  ip a
  ```
- **Show Only Active Interfaces:**  
  ```bash
  ip link show up
  ```
- **Assign IP Address to an Interface:**  
  ```bash
  sudo ip addr add 192.168.1.10/24 dev eth0
  ```
- **Bring Interface Up or Down:**  
  ```bash
  sudo ip link set eth0 up
  sudo ip link set eth0 down
  ```
- **Check Routing Table:**  
  ```bash
  ip route show
  ```

### **Examples:**  
```bash
ip a
ip link set eth0 up
ip addr add 10.0.0.5/24 dev eth0
ip route show
```

---

## **3. `ping` – Test Connectivity**  
- **Basic Connectivity Test:**  
  ```bash
  ping google.com
  ```
- **Specify Number of Packets to Send:**  
  ```bash
  ping -c 4 google.com
  ```
- **Change Packet Size:**  
  ```bash
  ping -s 1000 google.com
  ```

### **Examples:**  
```bash
ping -c 4 8.8.8.8
ping -s 1280 example.com
```

---

## **4. `traceroute` – Trace Route to Destination**  
- **Show Route and Latency:**  
  ```bash
  traceroute google.com
  ```
- **Specify Maximum Hops:**  
  ```bash
  traceroute -m 20 google.com
  ```
- **Use TCP Instead of ICMP:**  
  ```bash
  traceroute -T google.com
  ```

### **Examples:**  
```bash
traceroute example.com
traceroute -m 15 google.com
traceroute -T example.com
```

> **Note:** If `traceroute` is not installed, use `tracepath`.

---

## **5. `tracepath` – Trace Path to Destination (No Root Required)**  
- **Trace Path to a Host:**  
  ```bash
  tracepath google.com
  ```
- **Specify Maximum Hops:**  
  ```bash
  tracepath -m 30 example.com
  ```

### **Examples:**  
```bash
tracepath example.com
tracepath -m 20 google.com
```

---

## **6. `nslookup` – DNS Lookup**  
- **Find IP Address of a Domain:**  
  ```bash
  nslookup example.com
  ```
- **Find Domain of an IP Address (Reverse Lookup):**  
  ```bash
  nslookup 8.8.8.8
  ```
- **Specify DNS Server for Lookup:**  
  ```bash
  nslookup example.com 8.8.8.8
  ```

### **Examples:**  
```bash
nslookup google.com
nslookup 1.1.1.1
nslookup example.com 8.8.4.4
```

---

## **7. `dig` – Detailed DNS Lookup**  
- **Basic Domain Information:**  
  ```bash
  dig example.com
  ```
- **Get Only A Record (IP Address):**  
  ```bash
  dig example.com A
  ```
- **Reverse Lookup (Find Domain from IP):**  
  ```bash
  dig -x 8.8.8.8
  ```
- **Query Specific DNS Server:**  
  ```bash
  dig @8.8.8.8 example.com
  ```

### **Examples:**  
```bash
dig google.com
dig example.com A
dig -x 8.8.8.8
dig @8.8.4.4 example.com
```

---

## **8. `netstat` – Network Statistics**  
- **Show Active Connections:**  
  ```bash
  netstat -an
  ```
- **Show Listening Ports:**  
  ```bash
  netstat -l
  ```
- **Show Network Statistics:**  
  ```bash
  netstat -s
  ```

> **Note:** `netstat` is deprecated. Use `ss` instead.

---

## **9. `ss` – Socket Statistics (Modern Replacement for `netstat`)**  
- **Show Active Connections:**  
  ```bash
  ss -tuln
  ```
- **Show Established Connections:**  
  ```bash
  ss -tn state established
  ```
- **Show Listening Ports:**  
  ```bash
  ss -tul
  ```
- **Show Processes Using Ports:**  
  ```bash
  ss -tulnp
  ```

### **Examples:**  
```bash
ss -tuln
ss -tn state established
ss -tulnp
```

---

## **10. `curl` – Transfer Data from or to a Server**  
- **Fetch Web Page Content:**  
  ```bash
  curl example.com
  ```
- **Check HTTP Response Header:**  
  ```bash
  curl -I example.com
  ```
- **Send POST Request:**  
  ```bash
  curl -X POST -d "name=value" example.com/form
  ```

### **Examples:**  
```bash
curl google.com
curl -I example.com
curl -X POST -d "user=test&pass=1234" example.com/login
```

---

## **11. `wget` – Download Files from the Web**  
- **Download a File:**  
  ```bash
  wget http://example.com/file.zip
  ```
- **Download a File with Custom Name:**  
  ```bash
  wget -O custom_name.zip http://example.com/file.zip
  ```
- **Resume Interrupted Download:**  
  ```bash
  wget -c http://example.com/largefile.zip
  ```

### **Examples:**  
```bash
wget https://example.com/image.jpg
wget -O newfile.zip https://example.com/archive.zip
wget -c https://example.com/large_video.mp4
```

---

## **12. `hostname` – Display or Set Hostname**  
- **Show Current Hostname:**  
  ```bash
  hostname
  ```
- **Set a New Hostname (Temporary):**  
  ```bash
  sudo hostname new_hostname
  ```
- **Show IP Address of Hostname:**  
  ```bash
  hostname -I
  ```

### **Examples:**  
```bash
hostname
sudo hostname webserver01
hostname -I
```

---

## **13. `arp` – View or Manipulate ARP Cache**  
- **Show ARP Cache:**  
  ```bash
  arp -a
  ```
- **Delete an ARP Entry:**  
  ```bash
  sudo arp -d 192.168.1.1
  ```

---

## **Summary:**  
- **Interface Management:** `ifconfig`, `ip`  
- **Connectivity Tests:** `ping`, `traceroute`, `tracepath`  
- **DNS Lookups:** `nslookup`, `dig`  
- **Network Statistics:** `netstat`, `ss`  
- **File Transfers:** `curl`, `wget`  
- **ARP Cache Management:** `arp`
